# Dan Shipper

Founder of [[projects/every]], an AI-native media + software company. Co-author (with Claude) of the {{source:every-agent-native-guide}}Every "Agent-Native Architectures" guide{{/source}}, which lays out the [[concepts/agent-native]] framework being applied across Every's products.

## Relevance to this project

Shipper's framing of {{source:every-agent-native-guide}}files as the universal interface between humans and agents{{/source}} validates `sahana-wiki`'s markdown-first, no-database design. The guide's five principles (parity, granularity, composability, emergent capability, improvement over time) map directly onto how a librarian-style wiki should evolve.

His economic bet — {{source:parrott-four-apps}}that inference costs drop ~80% every few months{{/source}} — is the reason agent-native apps that look uneconomical today (Cora has had $1,500 days in inference) are worth building anyway.

## Demos

He's the demo-lead for Every's Agent Native Camp. The {{source:parrott-four-apps}}book-scanning app — point a camera at any page, get a spoiler-calibrated summary{{/source}} — is the canonical "look how little code this takes" example.
